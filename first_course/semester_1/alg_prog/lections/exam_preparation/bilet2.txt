===== Представление целых чисел в ЭВМ =====
>> Целые числа в ЭВМ представлены в двоичной системе счисления. Основные аспекты:
* Прямой код.
- старший бит указывает знак числа (0 - положительное; 1 - отрицательное), остальные биты хранят величину числа.
* Обратный код.
- используется для представления отрицательных чисел. Число представляется как инверсия всех битов положительного числа.
* Дополнительный код.
- Наиболее распростаненный способ представления целых чисел. Для отрицательных чисел это инверсия битов числа в прямом коде с добавлением единицы к младшему разряду.

* Используются фиксированные размеры памяти (например, 8, 16, 32, 64 бита).

Диапазон представления чисел: 
* Для знаковых 8-битных чисел: от -128 до 127.
* Для безнаковых 8-битных чисел: от 0 до 255.

Преимущества дополнительного кода:
* Простота арифметических операций.
* Единая обработка знаковых и безнаковых чисел.
* Устранение двух представлений нуля.

===== Основые типы данных в Си и операции над ними =====
>> Типы данных в Си:
1. Целочисленные типы:
* int (обычно 4 байта);
* short (обычно 2 байта);
* long (4 или 8 байт);
* unsigned (для беззнаковых чисел).

2. Вещественные типы:
* float (обычно 4 байта);
* double (обычно 8 байт);
*long double (может занимать 16 байт).

3. Символьный тип:
* char (1 байт);

4. Специальные типы:
* void (для функций без возвращаемого значения);
* указатели (хранят адреса).

===== Основные операции =====
* Арифметические: +, -, *, /, %;
* Логические: && (и), || (или), ! (не)
* Побитовые: & (и), | (или), ^ (исключающее или), ~ (инверсия), << (сдвиг влево), >> (сдвиг вправо)
* Операцие сравнения: ==, !=, <, >, <=, >=

===== Особенности операций над вещественными числами =====
1. Представление вещественных чисел:
* Используется стандарт IEEE 754:
- Число хранится в формате: (-1)^n * M * 2^e, где:
-- n - знак числа (0 - положительное; 1 - отрицательное);
-- M - мантисса (дробная часть);
-- e - порядок (экспонента)
* вещественные числа хранятся с ограниченной точностью, что может приводить к ошибкам округления.
2. Особенности операций:
* Погрешность вычислений: Результат может отличаться из-за округлений. Например, 0.1 + 0.2 != 0.3
* Ассоциативность и коммунитативность: Эти свойства могут нарушаться из-за ограниченной точности 
(например, (a + b) + c != a + (b + c))
* Проблемы переполнения и денормализации: Переполнение возникает, если число превышает диапазон, денормализация - при слишком малых значениях
3. Основные операции: +, -, *, /
* Специальные функции: pow, sqrt, sin, cos (в библиотеке math.h)
4. Практические рекомендации: 
* Избегать прямого сравнения вещественных чисел: a == b, использовать подход с учетом погрешности.

(-1)^n * m * 2^e
